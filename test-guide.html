curl -s -X POST http://localhost:4523/mock/8311458/api/v1/auth/login \

步骤 4：测试登录流程

4.1 密码登录

  1. 确认页面标题为"欢迎登录"，默认在"密码登录"标签
  2. 输入手机号 13800138000
  3. 输入密码 123456
  4. 勾选"我已阅读并同意用户协议与隐私政策"
  5. 点击"立即登录"
  6. 按钮显示 "登录中..." → 成功后弹出绿色 Toast → 自动跳转到 Lyncr 控制台

4.2 验证码登录

  1. 点击"验证码登录"标签
  2. 输入手机号 → 点击"获取验证码" → 按钮变成 60s 倒计时
  3. 输入任意 6 位验证码（Mock 不校验具体值）
  4. 勾选协议 → 点击"立即登录"
  5. 成功后跳转控制台

4.3 注册流程

  1. 点击底部"立即注册"
  2. 输入手机号 → 获取验证码 → 输入验证码
  3. 勾选协议 → 点击"立即注册"
  4. 成功后自动切换到验证码登录页并完成登录

请求链路说明


  浏览器                     Vite 代理                     Apifox
  ──────                     ────────                     ──────
  POST /api/v1/auth/login  → 转发到 mock/8311458/api  →  返回 Mock JSON
  (no CORS)                  /v1/auth/login
  • 前端 baseURL: '/api'，所有请求是相对路径，不跨域
  • Vite proxy 将 /api/* 转发到 http://localhost:4523/mock/8311458/api/*
  • axios 请求拦截器自动从 localStorage 读取 token 注入 Authorization 头
  • axios 响应拦截器统一解包 response.data，HTTP 错误转中文提示

常见问题

① 点击登录后提示"网络错误"

  1. 终端执行 lsof -i :4523 确认 Apifox 在运行
  2. 用步骤 2 的 curl 命令测试 API 是否通
  3. 如果 curl 通但前端不通 → 重启 pnpm dev

② 浏览器控制台报 CORS 错误

检查前端请求路径是否以 /api 开头（相对路径）。如果代码里写的是 http://localhost:4523/...（绝对路径），会跨域。正确写法是只用 /api/v1/auth/login。

③ 登录成功但没跳转

检查 Apifox Mock 响应里 data.token 字段是否存在。前端依赖 data.token 存储并跳转。

④ 如何从 Mock 切换到真实后端

修改 vite.config.ts 的 proxy target：

// Mock 模式（当前）
target: 'http://localhost:4523/mock/8311458/api'

// 真实后端
target: 'https://your-api-server.com/api'
