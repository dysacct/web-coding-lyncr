1. pnpm 包管理器和安装

pnpm 是什么

pnpm（performant npm）是一个高效的 Node.js 包管理器。它通过硬链接 + 符号链接的方式共享依赖，核心优势：

  • 磁盘空间极省：同一包的不同版本只存一份，多个项目共享
  • 安装极快：比 npm / yarn 快 2-3 倍
  • 严格的 node_modules：只有声明过的依赖才能被访问，杜绝幽灵依赖
  • 原生支持 monorepo：内置 workspace 功能

安装 pnpm

# 方式一：通过 npm 安装（推荐）
npm install -g pnpm

# 方式二：通过 Homebrew 安装（macOS）
brew install pnpm

# 方式三：通过 Corepack 启用（Node.js 16+ 内置）
corepack enable
corepack prepare pnpm@latest --activate

# 验证安装
pnpm --version

pnpm vs npm vs yarn 命令对照

操作pnpmnpmyarn

2. 用 pnpm 创建 Vue 3 + Vite 项目

创建项目

# 进入你的工作目录
cd ~/workspace

# 用 Vite 官方脚手架创建 Vue 3 + TypeScript 项目
pnpm create vite@latest my-vue-app -- --template vue-ts

# 进入项目
cd my-vue-app

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev

创建后的项目结构

├── index.html

pnpm-lock.yaml 注意事项

pnpm 生成的是 pnpm-lock.yaml，不是 package-lock.json。这个文件必须提交到 Git，确保团队所有人的依赖版本一致。

3. 下载和安装 UnoCSS

UnoCSS 是什么

UnoCSS 是一个即时按需的原子化 CSS 引擎。相比 Tailwind CSS：

  • 零配置启动，但高度可定制
  • 无解析、无 AST，直接正则匹配，构建更快
  • 通过 preset（预设） 系统完全兼容 Tailwind CSS 语法
  • 不生成几 MB 的初始 CSS，只生成你用到的样式

安装 UnoCSS

基础安装：

# 核心包 + Vite 插件（Vite 项目必装）
pnpm add -D unocss

# CSS 重置（类似 Tailwind 的 @tailwind base）
pnpm add -D @unocss/reset

常用预设（preset）：

# preset-wind3：Tailwind CSS v3 兼容层（最常用，安装一个即可拥有全部 Tailwind 语法）
pnpm add -D @unocss/preset-wind3

# preset-icons：图标支持（基于 Iconify，可使用 200+ 图标集的 20 万+ 图标）
pnpm add -D @unocss/preset-icons

# preset-attributify：允许用 HTML attribute 写样式
pnpm add -D @unocss/preset-attributify

# preset-typography：排版预设（类似 @tailwindcss/typography 插件）
pnpm add -D @unocss/preset-typography

# preset-rem-to-px：自动将 rem 转为 px
pnpm add -D @unocss/preset-rem-to-px

图标包（配合 preset-icons 使用）：

# 安装你需要的图标集，比如 Carbon Icons
pnpm add -D @iconify-json/carbon

# 其他常用图标集
pnpm add -D @iconify-json/mdi          # Material Design Icons
pnpm add -D @iconify-json/heroicons    # Tailwind Heroicons
pnpm add -D @iconify-json/ri           # Remix Icon
pnpm add -D @iconify-json/simple-icons # 品牌 Logo

一次安装全部推荐包

pnpm add -D unocss @unocss/reset @unocss/preset-wind3 @unocss/preset-icons @unocss/preset-attributify
pnpm add -D @iconify-json/carbon

4. 在项目中导入和配置 UnoCSS

第一步：修改 vite.config.ts，注册 UnoCSS 插件

// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import UnoCSS from 'unocss/vite'          // ← 导入 UnoCSS Vite 插件

export default defineConfig({
  plugins: [
    vue(),
    UnoCSS(),                              // ← 注册插件
  ],
})

UnoCSS 插件必须放在 Vue 插件之后

第二步：创建 uno.config.ts（UnoCSS 配置文件）

在项目根目录创建 uno.config.ts：

// uno.config.ts
import { defineConfig } from 'unocss'
import presetWind3 from '@unocss/preset-wind3'

export default defineConfig({
  presets: [
    presetWind3(),   // ← 启用 Tailwind CSS 兼容预设
  ],
})

第三步：在 main.ts 中导入 UnoCSS 样式

// src/main.ts
import { createApp } from 'vue'
import App from './App.vue'

// UnoCSS 的 CSS Reset（类似 @tailwind base）
import '@unocss/reset/tailwind.css'

// UnoCSS 按需生成的工具类样式
import 'virtual:uno.css'

createApp(App).mount('#app')

关于 virtual:uno.css

virtual:uno.css 是一个虚拟模块，由 UnoCSS Vite 插件在开发/构建时动态生成。它不是磁盘上的真实文件——插件会扫描你的源码，找到你用到的所有 class，然后只生成这些 class 对应的 CSS。这就是 UnoCSS "按需生成" 的核心机制。

5. preset-wind3：内置的 Tailwind CSS 兼容层

preset-wind3 是 UnoCSS 官方的 Tailwind CSS v3 兼容预设。安装了它，你就可以用所有你熟悉的 Tailwind CSS class：flex、grid、p-4、text-red-500、hover:bg-blue-600……全部可用，不需要单独安装 tailwindcss 包。

原生 Tailwind CSS 方式

# 需要安装这些
pnpm add -D tailwindcss postcss autoprefixer

# 需要 tailwind.config.js
module.exports = {
  content: ['./index.html', './src/**/*.{vue,ts}'],
  theme: { extend: {} },
  plugins: [],
}

# 主 CSS 文件中写三个 @tailwind 指令
@tailwind base;
@tailwind components;
@tailwind utilities;

UnoCSS + preset-wind3 方式

# 只需要这两个包
pnpm add -D unocss @unocss/preset-wind3

# uno.config.ts 一行配置
import { defineConfig } from 'unocss'
import presetWind3 from '@unocss/preset-wind3'

export default defineConfig({
  presets: [presetWind3()],
})

# main.ts 一行导入
import 'virtual:uno.css'

Tailwind 功能preset-wind3说明
布局 (flex, grid, block...)✅ 完全支持语法完全相同
间距 (p-, m-, gap-...)✅ 完全支持语法完全相同
颜色 (red-500, blue-600...)✅ 完全支持语法完全相同

6. UnoCSS 原生用法：class 书写、shortcuts、theme

在 Vue 组件中使用（和 Tailwind 语法一模一样）

<template>
  <!-- 所有 Tailwind class 直接可用 -->
  <div class="flex items-center justify-center min-h-screen bg-gray-900 p-4">
    <div class="max-w-md w-full bg-white rounded-2xl shadow-2xl p-8">
      <h1 class="text-2xl font-bold text-gray-800 mb-6">
        欢迎登录
      </h1>
      <button class="w-full py-3 bg-blue-600 hover:bg-blue-500
                       text-white font-medium rounded-xl
                       active:scale-98 transition-all">
        立即登录
      </button>
    </div>
  </div>
</template>

Shortcuts：自定义快捷类（替代 @apply）

Tailwind 里用 @apply 组合样式，UnoCSS 里用 shortcuts：

// uno.config.ts
export default defineConfig({
  presets: [presetWind3()],
  shortcuts: {
    // 简单映射：一个 key 对应一组 class
    'btn': 'py-2 px-4 font-semibold rounded-lg shadow-md',
    'btn-primary': 'btn bg-blue-600 text-white hover:bg-blue-500',
    'btn-danger': 'btn bg-red-600 text-white hover:bg-red-500',

    // 也支持动态规则（用正则）
    // 'btn-(.*)': 'btn bg-$1 text-white',  // 比如 btn-red → btn bg-red text-white
  },
})

在组件中直接使用：

<button class="btn-primary">登录</button>
<button class="btn-danger">删除</button>

Theme：自定义主题变量

// uno.config.ts
export default defineConfig({
  presets: [presetWind3()],
  theme: {
    colors: {
      // 扩展颜色板
      brand: {
        primary: '#6366f1',
        secondary: '#a855f7',
      },
    },
    // 自定义断点
    breakpoints: {
      tablet: '768px',
      desktop: '1024px',
    },
    // 自定义动画
    animation: {
      keyframes: {
        'slide-up': '{from{opacity:0;transform:translateY(16px)}to{opacity:1;transform:translateY(0)}}',
      },
      durations: {
        'slide-up': '0.3s',
      },
    },
  },
})

使用：text-brand-primary、tablet:text-lg、animate-slide-up

Rules：自定义原子类

// uno.config.ts
export default defineConfig({
  rules: [
    // 自定义 margin 规则：m-
    [/^m-(\d+)$/, ([, d]) => ({ margin: `${Number(d) * 4}px` })],
    // 自定义文字阴影
    ['text-shadow', { 'text-shadow': '0 2px 4px rgba(0,0,0,0.3)' }],
    // 自定义玻璃效果
    ['glass', {
      background: 'rgba(255,255,255,0.1)',
      'backdrop-filter': 'blur(10px)',
      border: '1px solid rgba(255,255,255,0.1)',
    }],
  ],
})

7. UnoCSS + Tailwind CSS 联合使用

联合使用的三种策略

策略 A（推荐）：只用 UnoCSS + preset-wind3，抛弃 Tailwind CSS

这是最推荐的方案。preset-wind3 完全兼容 Tailwind 语法，你写的 class 和 Tailwind 一模一样，但底层是 UnoCSS 引擎，速度快、按需生成、配置更灵活。你的项目就是这样做的：

// package.json（你当前项目的实际配置）
"devDependencies": {
  "unocss": "^66.7.0",
  "@unocss/preset-wind3": "^66.7.0",
  "@unocss/preset-icons": "^66.7.0",
  "@unocss/reset": "^66.7.0",
  // 注意：没有 tailwindcss 包！
}

策略 B：UnoCSS + Tailwind CSS 共存（两套体系并行）

如果你有一个已有的 Tailwind 项目，想逐步迁移到 UnoCSS，可以两套同时跑：

pnpm add -D unocss @unocss/preset-wind3
pnpm add -D tailwindcss postcss autoprefixer

关键配置：让两者不冲突

// uno.config.ts — UnoCSS 加上前缀，避免和 Tailwind 冲突
export default defineConfig({
  presets: [presetWind3()],
  prefix: 'u-',   // ← UnoCSS 的 class 加 u- 前缀
})

// tailwind.config.js — Tailwind 也加前缀
export default {
  content: ['./index.html', './src/**/*.{vue,ts}'],
  prefix: 'tw-',  // ← Tailwind 的 class 加 tw- 前缀
}

使用效果：

<!-- UnoCSS 的 class（加 u- 前缀）-->
<div class="u-flex u-items-center u-p-4 u-bg-blue-500">

<!-- Tailwind 的 class（加 tw- 前缀）-->
<div class="tw-flex tw-items-center tw-p-4 tw-bg-red-500">

<!-- 也可以混合使用 -->
<div class="tw-flex tw-p-4 u-bg-blue-500 u-text-white">

策略 C：UnoCSS 作为 Tailwind CSS 的插件（Tailwind 为主）

Vite 模式下 UnoCSS @unocss/postcss-extension 可作为 PostCSS 插件运行在 Tailwind 之前/之后，用 UnoCSS 补充 Tailwind 没有的功能（如纯 CSS 图标）。

这个方案比较小众，一般不需要。

三种策略对比

策略包体积构建速度学习成本适用场景
A：纯 UnoCSS最轻最快低新项目、想用 Tailwind 语法但追求性能
B：UnoCSS + Tailwind 共存较重中等中迁移中的项目、逐步替换
C：Tailwind 为主 + UnoCSS 补充最重最慢中高度依赖 Tailwind 插件生态的老项目

8. 进阶：图标、Variant Group、Attributify

图标（preset-icons）

安装：pnpm add -D @unocss/preset-icons @iconify-json/carbon

// uno.config.ts
import presetIcons from '@unocss/preset-icons'

export default defineConfig({
  presets: [
    presetWind3(),
    presetIcons({
      // 图标名映射为 class：i-carbon-{图标名}
      collections: {
        carbon: () => import('@iconify-json/carbon/icons.json').then(m => m.default),
      },
      scale: 1.2,
    }),
  ],
})

使用：

<!-- 直接用 class 渲染图标，无需 import 组件 -->
<span class="i-carbon-logo-wechat text-green-500 text-2xl" />
<span class="i-carbon-user-avatar text-blue-500" />
<span class="i-carbon-settings text-gray-400 hover:text-white" />

图标变成 纯 CSS 内联 SVG，不会增加 JS 体积，且可以像文字一样用 text-* 控制颜色和大小。

Variant Group（变体组）

安装：内置在 unocss 中，无需额外安装。

Variant Group 让你用括号将多个共享前缀的 class 分组：

<!-- 传统写法：重复写 hover: 前缀 -->
<div class="hover:bg-blue-600 hover:text-white hover:border-blue-600">

<!-- Variant Group：括号内共享 hover: 前缀 -->
<div class="hover:(bg-blue-600 text-white border-blue-600)">
// uno.config.ts — 需要在 config 中启用 transformer
import { transformerVariantGroup } from 'unocss'

export default defineConfig({
  transformers: [
    transformerVariantGroup(),   // ← 启用 Variant Group
  ],
})

更多用法：

<!-- 嵌套变体组 -->
<div class="sm:(p-4 flex gap-2) lg:(p-8 flex-col gap-4)">

<!-- 混合使用 -->
<button class="bg-blue-600 hover:(bg-blue-500 scale-105) active:scale-95">

Attributify 模式

安装：pnpm add -D @unocss/preset-attributify

让你直接在 HTML attribute 上写样式，不用 class 属性：

<!-- 传统 class 写法 -->
<button class="bg-blue-600 text-white py-3 px-6 rounded-xl font-bold">
  登录
</button>

<!-- Attributify 写法 -->
<button
  bg="blue-600"
  text="white"
  p="y-3 x-6"
  font="bold"
  border="rounded-xl"
>
  登录
</button>
// uno.config.ts
import presetAttributify from '@unocss/preset-attributify'

export default defineConfig({
  presets: [
    presetWind3(),
    presetAttributify(),   // ← 启用 Attributify
  ],
})

10. 多模式脚本：pnpm dev:h5 等命令的原理与配置

这些命令是什么

pnpm dev:h5、pnpm build:app、pnpm dev:mp-weixin 这类冒号后缀的命令，本质上是 package.json 里 scripts 的自定义命名。

它们的核心原理是 Vite 的 --mode 参数 + 多环境变量文件 + 多配置文件，从而让同一套代码能编译成不同平台的目标产物。

pnpm dev:h5
→
package.json scripts
→
vite --mode h5
→
加载 .env.h5
→
条件编译 → H5 产物

10.2 Vite --mode 参数的核心机制

--mode 是什么

vite --mode h5 告诉 Vite：使用 .env.h5 文件中的环境变量。

Vite 启动时，会自动加载以下 .env 文件（加载优先级从低到高）：

.env                  # 所有模式共享的变量
.env.local            # 本地覆盖，不提交 Git
.env.[mode]           # 特定模式的变量（如 .env.h5）
.env.[mode].local     # 特定模式的本地覆盖

创建多模式环境变量文件

# .env（所有模式共享）
VITE_APP_TITLE=我的应用
VITE_API_BASE_URL=https://api.example.com

# .env.h5（H5 模式专用变量）
VITE_PLATFORM=h5
VITE_API_BASE_URL=https://h5-api.example.com

# .env.app（App 模式专用变量）
VITE_PLATFORM=app
VITE_API_BASE_URL=https://app-api.example.com

# .env.mp-weixin（小程序模式专用变量）
VITE_PLATFORM=mp-weixin
VITE_API_BASE_URL=https://mp-api.example.com

变量名必须以 VITE_ 开头，才能在客户端代码中通过 import.meta.env.VITE_XXX 访问。

在代码中读取当前平台

// src/utils/platform.ts
export const platform = import.meta.env.VITE_PLATFORM || 'h5'

export const isH5 = platform === 'h5'
export const isApp = platform === 'app'
export const isMpWeixin = platform === 'mp-weixin'

// 根据平台选择不同实现
export function getStorage() {
  if (isApp)     return nativeStorage    // App 原生存储
  if (isMpWeixin) return wxStorage        // 微信小程序存储
  return localStorage                     // H5 浏览器存储
}
<!-- 在 Vue 模板中条件渲染 -->
<template>
  <!-- H5 端显示 -->
  <div v-if="isH5" class="flex items-center gap-2 p-4 bg-blue-500">
    这是 H5 移动端页面
  </div>

  <!-- App 端显示 -->
  <div v-if="isApp" class="p-4 bg-green-500">
    这是 App 端页面
  </div>

  <!-- 小程序端显示 -->
  <div v-if="isMpWeixin" class="p-4 bg-yellow-500">
    这是小程序端页面
  </div>
</template>

10.3 多 Platform 构建：不同 vite 配置文件

动态加载不同 Vite 配置

不同平台往往需要不同的插件、输出目录等。可以在 vite.config.ts 里根据 mode 动态切换：

// vite.config.ts
import { defineConfig, type UserConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import UnoCSS from 'unocss/vite'

export default defineConfig(({ mode }): UserConfig => {
  // mode 就是 --mode 参数的值
  console.log('当前构建模式:', mode)

  const baseConfig: UserConfig = {
    plugins: [vue(), UnoCSS()],
    resolve: { alias: { '@': '/src' } },
  }

  // 根据 mode 合并不同配置
  switch (mode) {
    case 'h5':
      return {
        ...baseConfig,
        base: '/h5/',
        build: { outDir: 'dist/h5' },
        define: { __PLATFORM__: JSON.stringify('h5') },
      }

    case 'app':
      return {
        ...baseConfig,
        base: '/app/',
        build: { outDir: 'dist/app' },
        define: { __PLATFORM__: JSON.stringify('app') },
      }

    case 'mp-weixin':
      return {
        ...baseConfig,
        build: { outDir: 'dist/mp-weixin' },
        define: { __PLATFORM__: JSON.stringify('mp-weixin') },
        // 小程序不需要 index.html
      }

    default:
      return baseConfig
  }
})

使用多个独立配置文件（更清晰）

复杂项目建议拆分成多个配置文件，共享一个 base 配置：

// vite.config.base.ts — 公共配置
import vue from '@vitejs/plugin-vue'
import UnoCSS from 'unocss/vite'
import type { UserConfig } from 'vite'

export const baseConfig: UserConfig = {
  plugins: [vue(), UnoCSS()],
  resolve: { alias: { '@': '/src' } },
}

// vite.config.h5.ts — H5 专用
import { defineConfig } from 'vite'
import { baseConfig } from './vite.config.base'

export default defineConfig({
  ...baseConfig,
  base: '/h5/',
  build: { outDir: 'dist/h5' },
})

// vite.config.app.ts — App 专用
import { defineConfig } from 'vite'
import { baseConfig } from './vite.config.base'

export default defineConfig({
  ...baseConfig,
  base: '/app/',
  build: { outDir: 'dist/app' },
  // 可能还会添加 App 专用插件，如 uniapp、电容插件等
})

脚本改为：

"scripts": {
  "dev:h5":  "vite --config vite.config.h5.ts",
  "dev:app": "vite --config vite.config.app.ts",
  "build:h5":  "vite build --config vite.config.h5.ts",
  "build:app": "vite build --config vite.config.app.ts",
}

10.4 CSS 条件编译

利用 UnoCSS 配合平台编译

你可以在 UnoCSS 配置里根据环境变量条件编译不同的样式：

// uno.config.ts
import { defineConfig } from 'unocss'
import presetWind3 from '@unocss/preset-wind3'

export default defineConfig({
  presets: [presetWind3()],
  theme: {
    // H5 移动端用 750px 设计稿的 rem 基准
    // App 端用 375px 设计稿
    // 可以在这里根据环境动态调整
  },
  // shortcuts 也可以根据平台不同
  shortcuts: {
    'safe-area': process.env.VITE_PLATFORM === 'app'
      ? 'pt-[env(safe-area-inset-top)]' // App 需要安全区
      : 'pt-0',                          // H5 不需要
  },
})

SCSS/CSS 条件编译（uni-app 风格）

/* style.css — uni-app 风格的条件编译 */
/* #ifdef H5 */
body {
  max-width: 750px;
  margin: 0 auto;
}
/* #endif */

/* #ifdef MP-WEIXIN */
body {
  /* 小程序没有 body，用 page */
}
page {
  background-color: #f5f5f5;
}
/* #endif */

如果使用 uni-app，条件编译由 @dcloudio/vite-plugin-uni 自动处理。
如果是原生 Vite 项目，推荐用 import.meta.env.VITE_PLATFORM 在 JS 中做判断。

10.5 实战：添加一套 H5 移动端模式到你的项目

Step 1：创建环境变量文件

.env
VITE_APP_TITLE=我的应用
.env.h5
VITE_PLATFORM=h5
VITE_APP_TITLE=我的应用-H5版
.env.app
VITE_PLATFORM=app
VITE_APP_TITLE=我的应用-App版

Step 2：在 package.json 添加脚本

{
  "scripts": {
    "dev":         "vite",
    "dev:h5":      "vite --mode h5",
    "dev:app":     "vite --mode app",
    "build":       "vite build",
    "build:h5":    "vite build --mode h5",
    "build:app":   "vite build --mode app",
    "preview":     "vite preview"
  }
}

Step 3：调整 vite.config.ts 根据 mode 切换行为

// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import UnoCSS from 'unocss/vite'

export default defineConfig(({ mode }) => ({
  plugins: [vue(), UnoCSS()],

  base: mode === 'h5' ? '/h5/' : '/',

  build: {
    outDir: `dist/${mode === 'h5' ? 'h5' : 'web'}`,
  },

  server: {
    // H5 移动端适配：手机扫码调试更方便
    host: '0.0.0.0',  // 局域网可访问
    port: mode === 'h5' ? 5175 : 5174,
  },
}))

Step 4：在代码中读取平台并条件执行

// src/App.vue
<script setup lang="ts">
const platform = import.meta.env.VITE_PLATFORM || 'web'

// 不同平台的初始化逻辑
if (platform === 'h5') {
  // H5 专用：设置 viewport、rem 适配等
  document.documentElement.style.setProperty('--safe-bottom', 'env(safe-area-inset-bottom)')
}
if (platform === 'app') {
  // App 专用：初始化原生 SDK、监听 backbutton 等
}
</script>

<template>
  <div class="min-h-screen">
    <router-view />
  </div>
</template>

10.7 速记：这些命令的本质

命令底层实际执行效果
pnpm dev vite 载入 .env + .env.development，启动开发服务器
pnpm dev:h5 vite --mode h5 载入 .env + .env.h5 + .env.h5.local，H5 模式开发
pnpm build vite build 载入 .env + .env.production，生产构建
pnpm build:h5 vite build --mode h5 载入 .env + .env.h5，构建 H5 产物到 dist/
pnpm preview vite preview 以生产模式预览 dist/ 目录的构建产物

关键要点

  1. 冒号 : 只是命名约定——你完全可以用 dev-h5、h5:dev 等任意名字，团队习惯用 动作:目标 格式。
  2. --mode X 让 Vite 加载 .env.X，你可以在 vite.config.ts 中通过 mode 参数读取。
  3. import.meta.env.VITE_XXX 可在任何源码中读取环境变量，VITE_ 前缀是强制要求。
  4. 不同平台的产物 通常写到 dist/{platform}/，避免互相覆盖。
  5. pnpm 的 run 可以省略：pnpm dev:h5 等价于 pnpm run dev:h5。
